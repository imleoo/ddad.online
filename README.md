## 📖 关于DDAD

DDAD（Document-Driven AI Development）是一套专为AI协作时代设计的开发方法论。当AI成为开发团队的一员时，DDAD通过结构化文档建立人机协作的精确契约，实现高效的团队协作。

### 核心原则

1. **规范即接口** (Spec as Interface) - 规范是AI的执行指令
2. **知识即代码** (Knowledge as Code) - 文档化的团队记忆
3. **协作即编排** (Collaboration as Orchestration) - 人机混合的工作流

## 📝 部署到GitHub Pages

### 方法一：通过GitHub网页界面

1. 将代码推送到GitHub仓库
2. 进入仓库的 Settings > Pages
3. 在 Source 下选择 `main` 分支
4. 点击 Save
5. 等待几分钟后访问 `https://yourusername.github.io/ddad.online`

### 方法二：使用自定义域名

1. 在仓库根目录创建 `CNAME` 文件，内容为你的域名（如 `ddad.online`）
2. 在域名提供商处配置DNS记录：
   ```
   类型: A
   名称: @
   值: 185.199.108.153
        185.199.109.153
        185.199.110.153
        185.199.111.153
   ```
   或使用CNAME记录：
   ```
   类型: CNAME
   名称: www
   值: yourusername.github.io
   ```
3. 等待DNS生效（可能需要几小时）
